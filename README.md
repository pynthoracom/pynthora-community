## ‚ùì How to Ask Questions

### Using GitHub Issues

1. **Check existing issues** to see if your question has already been asked
2. **Create a new issue** with a clear, descriptive title
3. **Use the question template** (if available) to structure your question
4. **Tag appropriately** with relevant labels (e.g., `question`, `help-wanted`, `beginner`)

### Question Template

When creating a new question, please include:

```markdown
## Question

[Your question here]

## Context

[Provide context about what you're trying to achieve]

## What I've tried

[Describe what you've already tried or researched]

## Additional Information

[Any other relevant information, screenshots, code snippets, etc.]
```

### Best Practices

- ‚úÖ **Be specific**: Provide clear details about your question
- ‚úÖ **Include context**: Explain what you're working on and why
- ‚úÖ **Show effort**: Describe what you've already tried
- ‚úÖ **Be respectful**: Remember that community members are volunteering their time
- ‚ùå **Don't ask duplicate questions**: Search first!
- ‚ùå **Don't be vague**: "It doesn't work" isn't helpful

## üí¨ How to Answer Questions

### Guidelines for Answers

1. **Be helpful and clear**: Provide detailed, actionable answers
2. **Include examples**: Code examples, screenshots, or step-by-step instructions
3. **Link to resources**: Share relevant documentation or resources
4. **Follow up**: Check back to see if the answer helped or if clarification is needed

### Answering Best Practices

- ‚úÖ **Be patient**: Not everyone has the same level of expertise
- ‚úÖ **Explain why**: Help others understand the reasoning, not just the solution
- ‚úÖ **Format code**: Use proper code blocks with syntax highlighting
- ‚úÖ **Mark as solved**: If you're the questioner, mark helpful answers
- ‚ùå **Don't be condescending**: Everyone starts somewhere
- ‚ùå **Don't spam**: Keep answers relevant and concise

## üìñ FAQ

### How do I mark a question as solved?

Close the issue and add a comment thanking those who helped. You can also add the `solved` label.

### Can I ask multiple questions in one issue?

It's better to create separate issues for different questions. This makes it easier to:

- Track and answer specific questions
- Search for solutions later
- Help others find relevant answers

### What if my question is off-topic?

Check the repository's scope and guidelines. If it's clearly off-topic, we may close it, but we'll try to redirect you to a better place.

### How long should I wait for an answer?

There's no guaranteed response time, but typically:

- Simple questions: A few hours to a day
- Complex questions: A few days
- Very specific/niche questions: May take longer

Be patient, and feel free to provide additional context if needed.
